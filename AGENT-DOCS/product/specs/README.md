# Specs

Use this folder for durable `SPEC-*` requirements docs when a change is larger
than a quick fix or has ambiguous behavior.

Specs own:

- problem statement
- scope and non-goals
- user-visible behavior
- acceptance criteria
- important constraints
- links to plans, ADRs, issues, and session logs

Keep agent-only specs in separate commits from upstreamable code/test changes
unless the user explicitly wants to propose the spec upstream.
